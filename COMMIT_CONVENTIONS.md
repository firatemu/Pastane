# Commit Conventions

Use Conventional Commits:
- `feat:` new capability
- `fix:` bug fix
- `docs:` documentation only
- `chore:` tooling or maintenance
- `refactor:` behavior-preserving code change
- `test:` tests only

Examples:
- `chore: scaffold monorepo quality tooling`
- `docs: add AI agent workflow guide`
- `chore: complete phase 01 foundation`

Keep commits focused, reviewable, and aligned to one phase or concern.
